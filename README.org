#+AUTHOR: aRenCoco

* Overview
Org-roam-organize is an Emacs package for managing and organizing Org-roam nodes and their references. It provides a minor mode with commands for structuring knowledge bases, including node management, tag handling, backlink completion, and content organization.

* Features
The package includes the following main features:

** 1. Node Management
- Move headlines and their corresponding node files between directories (deprecated)
- Delete headlines and their associated files completely
- Update node tags automatically during operations

** 2. Tag Processing
- Automatic tag replacement (eg., =:idea:= to =:zettel:=)
- Tag-based node counting and statistics
- File tag updates across the knowledge base

** 3. Backlink Management
- Automatic insertion of missing backlinks for literature nodes
- Citation-based reference completion
- Link consistency maintenance

** 4. Statistical Functions
- Count nodes with specific tags
- Update node statistics in MOC files
- Maintain dynamic counters in properties

** 5. MOC Management
- Create MOC nodes based on templates
- Dynamic MOC discovery via tags and properties

** 6. File Organization
- Move node files to structured directory layouts (deprecated)
- Maintain directory organization based on node types (deprecated)
- ID-based directory creation and management (deprecated)

* Requirements
This package depends on the following:

- Emacs 30.1 or later
- Org 9.5 or later
- Org-roam 2.2.0 or later

* Configuration
Before using this package, you need to configure the following variables:

- =org-roam-organize-directory=: Root directory for org-roam-organize
- =org-roam-organize-directory-p=: Allow running outside root directory
- =org-roam-organize-moc-directory=: Directory for MOC (Map of Contents) files
- =org-roam-organize-moc-tag=: Tag used to identify MOC files
- =org-roam-organize-moc-managed-tag-property=: Property to store managed tag
- =org-roam-organize-moc-managed-node-count-property=: Property to store node count
- =org-roam-organize-top-moc-file=: Path to the top-level MOC file
- =org-roam-organize-tag-title-alist=: Association list mapping tags to MOC TITLEs
- =org-roam-organize-capture-template=: Org roam capture template for capture MOCs

The following fixed directory and file-moving variables are deprecated and kept for compatibility:

- =org-roam-organize-fleeting-directory=: Deprecated directory for temporary nodes
- =org-roam-organize-permanent-directory=: Deprecated directory for permanent nodes
- =org-roam-organize-move-target-directory=: Deprecated target directory for node movement
- =org-roam-organize-move-source-tag=: Deprecated source tag for automatic replacement
- =org-roam-organize-move-target-tag=: Deprecated target tag for automatic replacement
- =org-roam-organize-move-target-directory-id-or-not=: Deprecated option for ID-based directories
- =org-roam-organize-move-target-filename-id-or-not=: Deprecated option for using ID as filename

* Keybindings
This package does not install default keybindings. The commands are intended to be bound in user configuration.

** Suggested Global Keybindings
The following =C-c o= keybindings are optional user configuration examples:

- =C-c o o=: Toggle =org-roam-organize-mode=
- =C-c o c=: Check setup variables and runtime capabilities
- =C-c o h d=: Delete current headline and its corresponding node file
- =C-c o m m=: Jump to the top-level MOC
- =C-c o m u=: Update all MOC node statistics
- =C-c o r c=: Complete missing backlinks for literature nodes

Deprecated command:

- =org-roam-organize-headline-move=: Deprecated command for moving the current headline and its corresponding node file

** Suggested Command Aliases
The following aliases are optional user configuration examples:

* Usage
1. Configure the required variables in your Emacs configuration
2. Enable =org-roam-organize-mode= with =M-x org-roam-organize-mode= or your configured toggle key
3. Use the keybindings to organize your Org-roam knowledge base
4. Run =org-roam-organize-check-setup= to verify your configuration and runtime capabilities

An example of configuration is shown here.
#+BEGIN_SRC emacs-lisp
  (use-package org-roam-organize
    :straight (:host github :repo "ren-lingyu/org-roam-organize")
    :after org-roam
    :custom
    (org-roam-organize-directory org-roam-directory)
    (org-roam-organize-moc-directory (expand-file-name "./moc/" org-roam-directory))
    (org-roam-organize-moc-tag "map")
    (org-roam-organize-top-moc-file (expand-file-name "./moc/maps.org" org-roam-directory))
    (org-roam-organize-directory-p t)
    (org-roam-organize-tag-title-alist '(("map" . "Maps")
                                         ("zettel" . "Permanent")
                                         ("ref" . "Literature")
                                         ("idea" . "Fleeting")
                                         ("note" . "Note")
                                         ("blog" . "Blog")))
    (org-roam-organize-moc-managed-tag-property "MOC_MANAGED_TAG")
    (org-roam-organize-moc-managed-node-count-property "MOC_MANAGED_NODE_COUNT")
    (org-roam-organize-capture-template `("m" "map of contents" plain "%?"
                                          :if-new (file+head
                                                   "moc/${slug}.org"
                                                   ,(concat
                                                     ":PROPERTIES:\n"
                                                     ":MOC_MANAGED_TAG: ${moc_managed_tag}\n"
                                                     ":MOC_MANAGED_NODE_COUNT:\n"
                                                     ":END:\n"
                                                     "#+TITLE: ${title}\n"
                                                     "#+AUTHOR: aRenCoco\n"
                                                     "#+DATE: " (format-time-string "<%Y-%m-%d %A %H:%M:%S %z>") "\n"
                                                     "#+FILETAGS: :map:\n"
                                                     "#+DESCRIPTION:\n"))
                                          :unnarrowed t))
    :config
    (org-roam-organize-create-directory)
    :bind
    (("C-c o o" . org-roam-organize-mode)
     ("C-c o c" . org-roam-organize-check-setup)
     ("C-c o h d" . org-roam-organize-headline-delete)
     ("C-c o m m" . org-roam-organize-goto-map-of-maps)
     ("C-c o m u" . org-roam-organize-update-mocs)
     ("C-c o r c" . org-roam-organize-ref-backlink-complete))
    :hook (after-init-hook . org-roam-organize-mode))

#+END_SRC

* Implementation Details
The package uses Org-roam's database queries to manage node relationships and maintain consistency across the knowledge base. It includes configuration validation, runtime capability validation, and warning messages for common setup problems.

The mode includes a startup hook that validates configuration variables and ensures the Emacs session is started in the appropriate directory.
